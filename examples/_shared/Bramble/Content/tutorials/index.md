These lessons build on each other. Start at the top with a fresh install and work
down; by the end you will have written a CLI tool, published a package, and stood
up a small web service. Each lesson ends with running code, not a cliffhanger.

If you only have ten minutes, do [Install Bramble](xref:bramble.tutorials.install-bramble)
and [Your first program](xref:bramble.tutorials.your-first-program) — that is
enough to start poking at things in the REPL.

## The path

1. [Install Bramble](xref:bramble.tutorials.install-bramble)
2. [Your first program](xref:bramble.tutorials.your-first-program)
3. [Variables and values](xref:bramble.tutorials.variables-and-values)
4. [Control flow](xref:bramble.tutorials.control-flow)

11. [Packaging with Thicket](xref:bramble.tutorials.packaging-with-thicket)
12. [A web service](xref:bramble.tutorials.a-web-service)

## What you need

- A machine running macOS, Linux, or Windows
- A terminal you are comfortable in
- About an afternoon for the whole path, or one lesson per coffee break

You do not need prior experience with statically-typed languages. Where Bramble
does something unusual — no `null`, errors as values — the lesson explains just
enough to keep moving and links to the [explanations](xref:bramble.explanation.index)
for the full story.
