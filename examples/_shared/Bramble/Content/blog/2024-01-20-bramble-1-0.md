One year ago today I published the first commit with an actual `bramble` binary that could run a Hello World program. Today we're shipping 1.0, and I find myself equal parts proud and slightly overwhelmed by how far the language has come.

Bramble 1.0 is a stability commitment. Starting today, code that compiles under 1.0 will continue to compile under any future 1.x release. The bytecode format is stable. The standard library's public API is stable. If we break any of that, it's a bug, not a feature.

## What's in 1.0

The full changelog is long. Here's what I think matters most:

**The language is complete enough to use seriously.** Pattern matching, `Option`, `Result`, closures, the ownership model, the module system, generics — all of it works together the way it should. We've been running Bramble inside Fernweb's API gateway for three months and it has been unremarkable in the best possible way.

**The standard library is coherent.** We made hard cuts. Things that didn't belong in the standard library were moved to Thicket packages. The core is small and the docs cover all of it.

**The error messages are good now.** This took longer than the features. A compiler that's right but incomprehensible isn't useful. We rewrote the diagnostics engine twice. I think we got there.

```bramble
// This is the kind of error message 1.0 gives you:
//
// error[E014]: cannot borrow `config` as mutable while an immutable borrow is active
//   --> src/server.br:42:5
//    |
// 38 |     let host = &config.host
//    |                 ------ immutable borrow occurs here
// 42 |     config.reload()
//    |     ^^^^^^ mutable borrow occurs here
// 43 |     print(host)
//    |           ---- immutable borrow later used here
```

**Thicket has over 200 packages.** The ecosystem grew faster than I expected. That's not our doing — it's the community's.

## What 1.0 is not

It's not the end. Async and tasks are coming in 1.2. There's a 2.0 preview in planning that explores some ideas we had to defer. The [design philosophy](xref:bramble.explanation.design-philosophy) document explains which constraints we held firm and which we left flexible for future versions.

1.0 also doesn't mean "finished" in the sense that we're done listening. The community survey is coming later this year. Your feedback directly shapes what we work on.

## Thank you

There's a list of contributors in the release notes that runs to nearly 400 names. Every bug report, every PR, every forum thread where someone helped a stranger debug their borrow checker error — it all went into this release.

A few people deserve specific mention: Maple Okafor designed the type system with extraordinary care, Dr. Hazel Mbeki built the borrow checker from scratch, Rowan Vance kept the package ecosystem honest, and Juniper Sato made the tooling feel like a real product. They didn't have to bet on this language. I'm glad they did.
