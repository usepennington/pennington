# Penn Changelog

## Task 1: Project Scaffolding + UrlPath Value Type
- Created project files (Penn.csproj, Penn.Tests.csproj, GlobalUsings)
- Added C# 15 union type polyfills in Infrastructure/UnionPolyfills.cs
- Implemented UrlPath readonly record struct in Routing/UrlPath.cs
  - `/` operator for path combining
  - EnsureLeadingSlash, EnsureTrailingSlash, RemoveTrailingSlash, RemoveLeadingSlash
  - Matches() for tolerant URL comparison (trailing slash, index.html variants, case-insensitive)
  - Implicit conversion from string
- Added comprehensive UrlPath tests

## Task 2: FilePath + ContentRoute + ContentRouteFactory
- Implemented FilePath readonly record struct (Routing/FilePath.cs)
  - `/` operator, Extension, FileName, FileNameWithoutExtension
- Implemented ContentRoute sealed record (Routing/ContentRoute.cs)
  - CanonicalPath, OutputFile, SourceFile, Locale
  - Derived: NavigationPath, WithBaseUrl(), AbsoluteUrl(), IsDefaultLocale
- Implemented ContentRouteFactory static class (Routing/ContentRouteFactory.cs)
  - FromMarkdownFile, FromRazorPage, FromUrl, FromCustom, ForRedirect
  - All URL logic centralized in factory methods
- Added comprehensive tests for all routing types

## Task 3: Front Matter System
- Implemented IFrontMatter core interface (FrontMatter/IFrontMatter.cs)
- Implemented 8 capability interfaces (FrontMatter/Capabilities.cs)
  - IDraftable, ITaggable, IRedirectable, ISectionable
  - ICrossReferenceable, IOrderable, IDescribable, IDateable
- Implemented DocFrontMatter convenience record (FrontMatter/DocFrontMatter.cs)
- Implemented BlogFrontMatter convenience record (FrontMatter/BlogFrontMatter.cs)
- Added tests for interface implementation, defaults, capability checking

## Task 4: Pipeline Support Types + ContentSource Union

- Implemented SearchIndexDocument record (Search/SearchIndexDocument.cs)

- Implemented ProgrammaticContent union (TextProgrammaticContent, BinaryProgrammaticContent)
- Implemented IProgrammaticContentGenerator interface
- Added tests for all types, union conversions, and exhaustive pattern matching

## Task 5: ContentItem Union

## Task 11: Code Highlighting Service
- Implemented HighlightingService with priority-based highlighter dispatch

- Tests cover priority resolution, fallback, multi-language scenarios

## Task 12: Navigation Tree Builder
- Implemented NavigationBuilder with BuildTree() and BuildNavigationInfo()
- Tree building from flat ContentTocItem list using HierarchyParts
- Selection state and ancestor expansion from current route
- Depth-first prev/next navigation
- Breadcrumb trail computation
- Tests cover hierarchy, ordering, selection, prev/next, breadcrumbs

## Task 13: Build Report Output
- Added WriteTo(TextWriter) and ToFormattedString() to BuildReport
- Structured output: summary, errors with source files, warnings, broken links
- Tests cover clean builds, error/warning/broken-link scenarios, formatting

## Task 14: Front Matter Parsing
- Added YamlDotNet package dependency
- Implemented FrontMatterParser with YAML extraction and deserialization
- FrontMatterResult<T> carries parsed metadata + remaining markdown body
- Supports camelCase YAML properties, ignores unknown fields
- Tests cover DocFrontMatter, BlogFrontMatter, missing/empty front matter, tags, dates

## Task 15: Search Index + Feed Builders
- Implemented SearchIndexBuilder with HTML stripping for plain-text body
- Implemented SitemapBuilder with draft exclusion and absolute URLs
- Implemented RssFeedBuilder with date sorting and draft/undated exclusion
- All builders use capability interface checks (IDraftable, IDateable, etc.)
- Tests cover happy path, draft exclusion, date handling, HTML stripping

## Task 16: Link Verification Service
- Implemented LinkVerificationService for static link analysis
- Extracts href and src attributes from rendered HTML via regex
- Classifies links as ValidLink, BrokenLinkResult, or ExternalLink
- Handles query strings, anchors, trailing slashes, external URLs
- Tests cover valid/broken/external links, images, mixed pages

## Task 17: Content Pipeline Implementation
- Implemented ContentPipeline orchestrating Discover → Parse → Render → Generate
- IContentParser and IContentRenderer interfaces for pluggable stage handlers
- FailedItem propagation: errors flow through entire pipeline untouched
- Draft filtering at Generate stage (IDraftable → SkippedPages)
- Exception safety: parser/renderer exceptions wrapped into FailedItem
- RunAsync() convenience for full end-to-end pipeline execution
- Tests cover happy path, error propagation, drafts, exceptions, multiple services
