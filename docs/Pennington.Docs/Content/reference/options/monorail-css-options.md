> **In this page.** `ColorScheme`, `CustomCssFrameworkSettings`, `ExtraStyles`, and `ContentPaths` on `MonorailCssOptions`, plus the two built-in `IColorScheme` implementations (`NamedColorScheme`, `AlgorithmicColorScheme`).
>
> **Not in this page.** The MonorailCSS generator internals — how classes are collected, how the framework is constructed, and how palettes are derived — see [MonorailCSS integration](/explanation/rendering/monorail-css).

## Summary

_**One sentence: what it is.** The options class supplied to `services.AddMonorailCss(Func<IServiceProvider, MonorailCssOptions>?)` that drives the MonorailCSS stylesheet generated by `UseMonorailCss`._
_**One sentence: where it lives.** Declared in namespace `Pennington.MonorailCss` at `src/Pennington.MonorailCss/MonorailCssOptions.cs`, alongside the `IColorScheme` contract and its two built-in implementations._

## Properties

_Alphabetical. One-sentence lookup-shape descriptions only._

| Name | Type | Default | Description |
|---|---|---|---|
| `ColorScheme` | `IColorScheme` | `NamedColorScheme` with `Blue` primary, `Purple` accent, `Cyan` tertiary-one, `Pink` tertiary-two, and `Slate` base | _One-sentence: palette applied to the MonorailCSS theme via `IColorScheme.ApplyToTheme`, exposing the `primary`, `accent`, `tertiary-one`, `tertiary-two`, and `base` utility prefixes._ |
| `ContentPaths` | `string[]` | `[]` | _One-sentence: extra file paths (relative to the web root) scanned at startup for CSS class usage, ensuring classes referenced only in client-side JS or other non-HTML files reach the generated stylesheet._ |
| `CustomCssFrameworkSettings` | `Func<CssFrameworkSettings, CssFrameworkSettings>` | identity (`settings => settings`) | _One-sentence: hook that receives the fully-populated `CssFrameworkSettings` (theme, applies, prose customization) and returns a modified copy, allowing advanced MonorailCSS customization after Pennington's defaults are applied._ |
| `ExtraStyles` | `string` | `string.Empty` | _One-sentence: raw CSS prepended verbatim to the generated stylesheet — used for `@font-face` rules, site-wide overrides, or anything outside MonorailCSS's utility generation._ |

## Color-scheme types

_The TOC bundles both built-in `IColorScheme` implementations on this page; each gets its own declaration fence and sub-table so the page remains one coherent unit. The `IColorScheme` contract itself is a single-method interface — `Theme ApplyToTheme(Theme theme)` — and custom implementations are out of scope here (see the Explanation page)._

### `NamedColorScheme`

```csharp:xmldocid
T:Pennington.MonorailCss.NamedColorScheme
```

_Color scheme that maps five named Tailwind color palettes (from `MonorailCss.Theme.ColorNames`) onto the Pennington utility prefixes. All five names are `required`._

| Name | Type | Description |
|---|---|---|
| `AccentColorName` | `string` (required) | _One-sentence: name from `MonorailCss.Theme.ColorNames` mapped to the `accent` palette._ |
| `BaseColorName` | `string` (required) | _One-sentence: name from `MonorailCss.Theme.ColorNames` mapped to the `base` palette — drives neutral surfaces, borders, and body text._ |
| `PrimaryColorName` | `string` (required) | _One-sentence: name from `MonorailCss.Theme.ColorNames` mapped to the `primary` palette._ |
| `TertiaryOneColorName` | `string` (required) | _One-sentence: name from `MonorailCss.Theme.ColorNames` mapped to the `tertiary-one` palette._ |
| `TertiaryTwoColorName` | `string` (required) | _One-sentence: name from `MonorailCss.Theme.ColorNames` mapped to the `tertiary-two` palette._ |

### `AlgorithmicColorScheme`

```csharp:xmldocid
T:Pennington.MonorailCss.AlgorithmicColorScheme
```

_Color scheme that generates palettes algorithmically from a single primary hue, using a caller-supplied generator function to derive the accent and two tertiary hues._

| Name | Type | Default | Description |
|---|---|---|---|
| `BaseColorName` | `string` | `ColorNames.Gray` | _One-sentence: name from `MonorailCss.Theme.ColorNames` mapped to the `base` palette; the other four palettes are generated, not named._ |
| `ColorSchemeGenerator` | `Func<int, (int, int, int)>` | `primary => (primary + 180, primary + 90, primary - 90)` | _One-sentence: function that takes the primary hue and returns a tuple of `(accent, tertiaryOne, tertiaryTwo)` hues, each fed into `ColorPaletteGenerator.GenerateFromHue`._ |
| `PrimaryHue` | `int` (required) | — | _One-sentence: seed hue in the `0..360` range from which the primary palette is generated._ |
