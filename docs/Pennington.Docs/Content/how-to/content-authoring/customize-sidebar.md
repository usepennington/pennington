## When to use this

_Two sentences. Frame the arrival state: the reader has a DocSite with multiple markdown pages grouped under subfolders, the sidebar is auto-generated by `NavigationBuilder` from that folder tree, and now they want to nudge the layout — reorder siblings, make one page the section landing, rename the section header, or hide a page entirely. Clarify that this page is about front-matter and folder-name knobs; anything that requires swapping the component itself is the extensibility topic._

## Assumptions

_Keep to 3 bullets. The reader must already have at least one subfolder under their area so grouping exists to customize._

- You have a Pennington DocSite with markdown under `Content/<area>/` and at least one subfolder (the subfolder is what creates a sidebar group — see [Work with front matter](xref:how-to.content-authoring.front-matter) if not)
- Your pages use `DocSiteFrontMatter` or another type that implements `IOrderable` + `ISectionable`
- You understand the basics of `order:` and `isDraft:` — if not, start with [Manage drafts, tags, and ordering](xref:how-to.content-authoring.drafts-tags-ordering)

To copy a working setup, see [`examples/DocSiteKitchenSinkExample`](https://github.com/usepennington/pennington/tree/main/examples/DocSiteKitchenSinkExample) — `Content/main/customize-sidebar.md` exercises the same keys. Do not walk through the whole example — this page is a recipe, not a tour.

## Steps

_Four verb-first steps, one per TOC bullet. Each step ends with a one-line "what happens" on the sidebar output. Keep prose under two sentences per step. ### 1. Reorder pages within a section

_Two sentences. Lower `order:` values sort earlier inside a section; ties break alphabetically on `Title`. Recommend the 10/20/30 spacing convention so later inserts get room to land between siblings without renumbering every file._

```yaml
---
title: Install
order: 10
---
```

_Backing symbol on the DocSite front-matter record:_

```csharp:xmldocid
P:Pennington.DocSite.DocSiteFrontMatter.Order
```

### 2. Promote a page to be the section landing

_Two to three sentences. Name the file `index.md` inside the section subfolder (e.g. `Content/main/widgets/index.md`); Pennington routes it at the subfolder URL and `NavigationBuilder` surfaces it as the section's lead entry rather than a separate child. Give it a low `order:` (typically `10`) so it also pulls the whole section upward, since the section's aggregate sort key is the minimum `order:` of its direct children. Note that the `section:` wording in the TOC maps to the folder layout, not a front-matter key — the key is `sectionLabel:`, which is a display-only override covered in step 3. ### 3. Override the displayed section title

_Three sentences. Pennington's default sidebar section header comes from the folder name, kebab-case converted to title case by `NavigationBuilder.FormatSectionTitle` (`getting-started` → "Getting Started"). To change what the sidebar prints, rename the folder — that is the only knob that moves the sidebar header text, because the front-matter `sectionLabel:` key populates `ContentTocItem.SectionLabel` / `NavigationInfo.SectionName` (used for current-page context and breadcrumbs), **not** the sidebar group label. FLAG: the docs-toc premise "overriding the displayed section title" via front-matter is inaccurate against current code — this step should be framed as "rename the folder to change the sidebar header; set `sectionLabel:` to change the page-context label surfaced on `NavigationInfo.SectionName`." Escalate if the TOC line should be rewritten instead of the step._

```yaml
---
title: Install
sectionLabel: Quick Start
---
```

_Backing symbol for the front-matter key (display-only override, not the sidebar header):_

```csharp:xmldocid
P:Pennington.DocSite.DocSiteFrontMatter.SectionLabel
```

### 4. Hide a page from the sidebar

_Two sentences. Set `isDraft: true` to keep the page compiled (so `xref:` links still resolve) while dropping it from the sidebar, the search index, and `llms.txt` — `MarkdownContentService` skips draft items in both `DiscoverAsync` and `GetContentTocEntriesAsync`. A page with `redirectUrl:` is also omitted from the sidebar regardless of other keys, since the engine treats redirects as transport hops, not content._

```yaml
---
title: Work in progress
isDraft: true
---
```

_Backing symbol on `IFrontMatter` (the draft key is a default member, not specific to DocSite):_

```csharp:xmldocid
P:Pennington.FrontMatter.IFrontMatter.IsDraft
```

## Verify

_Terse. Three bullets, one per outcome so the reader can confirm each independently._

- Run `dotnet run`; reordered pages now appear in ascending `order:` inside their section, and the section itself moves when its minimum-child order changes
- The section subfolder's `index.md` lands at `/<area>/<section>/` and renders as the section's lead entry in the sidebar
- The drafted page's URL returns 404 (or a redirect page returns its meta-refresh) and the entry is absent from the sidebar on the next reload
